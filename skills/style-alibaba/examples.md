# 阿里巴巴风格示例 (Alibaba Style Examples)

## 示例一：电商运营周报

### 用户输入

```
我是阿里电商运营，这周的情况：
- 月度GMV目标5000万，本周累计完成3800万（本月第三周，达成率76%）
- 上线了直播专场活动，3场直播总观看150万，带货GMV 680万
- 转化率从2.1%提升到2.8%，主要是优化了商品详情页
- 客诉量这周涨了15%，主要是物流延迟，已经联系仓储排查
- 价值观方面：有个商家想做虚假促销，我拒绝了，虽然短期会损失50万GMV
- 下周要冲最后一周，差距1200万，计划加3场直播+一个限时折扣活动
```

### ManageUp 输出

```markdown
# 电商运营团队周报 2026.03.16

## 核心结论

月度 GMV 达成 76%（3800 万/5000 万），直播带货贡献 680 万。客诉量上涨 15% 需关注物流问题。最后一周差距 1200 万，计划通过加场直播 + 限时折扣冲刺。

## KPI 达成

| KPI | 月度目标 | 当前完成 | 达成率 | 本周增量 |
|-----|---------|---------|--------|---------|
| GMV | 5000 万 | 3800 万 | 76% | +680 万 |
| 转化率 | 3.0% | 2.8% | 93% | +0.7pp |
| 客诉率 | <2% | 2.3% | 未达 | +15% ⚠️ |

## 本周关键动作

- **直播专场 ×3**：总观看 150 万，带货 GMV 680 万（场均 227 万）→ 本周 GMV 核心贡献
- **商品详情页优化**：转化率从 2.1% 提升至 2.8%（+33%）→ 缩小转化率目标差距至 0.2pp

## 价值观实践

- **客户第一**：某商家提出虚假促销方案（预计带来 50 万 GMV），评估后认定损害消费者权益，拒绝执行。短期损失 50 万 GMV，但维护了平台信任，避免潜在客诉和平台处罚风险。

## 问题与复盘

- **目标差距**：月度 GMV 差 1200 万（24%），剩余 1 周
- **根因分析**：前两周直播场次不足（仅 2 场），直播启动晚于计划 5 天，导致 GMV 积累慢
- **客诉上涨 15%**：主因物流延迟，已联系仓储团队排查瓶颈，预计周三前给出改善方案

## 下周冲刺计划

| 事项 | 目标 | 策略 | 所需资源 |
|------|------|------|---------|
| 加场直播 ×3 | GMV +900 万 | 头部主播 + 爆品组合 | 主播档期确认 |
| 限时折扣活动 | GMV +300 万 | 48 小时限时折扣 | 商家备货确认 |
| 物流问题解决 | 客诉率降至 <2% | 仓储排查 + 备选快递 | 仓储团队配合 |
```

## 示例二：述职片段——价值观维度

### 用户输入

```
帮我写述职的价值观部分，阿里风格：
- 我负责搜索算法团队，今年做了个大的架构迁移
- 迁移过程中发现新架构有个边缘场景会影响3%的长尾商家搜索排名
- 团队建议先上线再修，因为排期紧张
- 我坚持先修再上，多花了2周，但确保了所有商家的搜索体验一致
- 最终按时交付了（加班赶回来的），新架构搜索相关性提升12%，QPS提升40%
- 还帮隔壁推荐团队做了技术分享，他们基于我们的方案优化了推荐模块
```

### ManageUp 输出

```markdown
## 价值观维度

### 客户第一

- **场景**：搜索架构迁移过程中，发现新架构存在边缘场景缺陷，影响约 3% 的长尾商家搜索排名准确性。团队建议先上线后修复以保排期。
- **选择**：坚持先修复再上线，额外投入 2 周修复边缘场景。通过团队加班赶回排期。
- **结果**：所有商家搜索体验一致性保障，无一起商家投诉。新架构最终按期交付，搜索相关性 +12%，QPS +40%。

### 此时此刻，非我莫属

- **场景**：推荐团队在优化推荐模块时遇到技术瓶颈，与搜索架构有共通性。
- **选择**：主动发起跨团队技术分享，输出搜索架构迁移方案和经验。
- **结果**：推荐团队基于分享方案优化了推荐模块，减少了约 3 周的重复探索时间。

### 今天最好的表现是明天最低的要求

- **场景**：新架构上线后 QPS 提升 40% 已达目标。
- **选择**：未满足于当前成果，已启动下一阶段优化规划，目标 QPS 再提升 20%。
- **结果**：下阶段技术方案已完成评审，计划 Q2 落地。
```
