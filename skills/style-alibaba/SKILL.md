# 阿里巴巴汇报风格 (Alibaba Reporting Style)

继承 `manage-up-core` 五大原则。本技能将报告适配阿里巴巴的双轨制文化——业绩与价值观并重，用事实说话。

---

## 触发场景

- 用户提到"阿里风格"、"阿里巴巴"、"Alibaba"
- 用户提到"六脉神剑"、"双轨制"、"价值观考核"
- 用户要求按阿里格式写周报、述职或绩效
- 用户背景是阿里巴巴或类似价值观驱动型组织

---

## 必填输入 (Required Inputs)

生成报告前，向用户收集以下信息：

```
为了写出符合阿里风格的报告，请提供以下信息：

1. **核心 KPI**：你的年度/季度/月度 KPI 是什么？完成情况如何？
   例：月度 GMV 目标 5000 万，当前完成 4200 万（84%）

2. **关键项目/事项**：本周做了哪些重要的事？结果是什么？
   例：上线了新的推荐算法，转化率提升了 15%

3. **价值观体现**（重要）：有没有体现阿里价值观的具体事例？
   例：为了客户体验，主动放弃了一个短期收益但伤害用户的方案
   阿里新六脉神剑：客户第一、员工第二、股东第三 / 因为信任所以简单 / 唯一不变的是变化 / 今天最好的表现是明天最低的要求 / 此时此刻非我莫属 / 认真生活快乐工作

4. **问题与复盘**：遇到什么问题？差距在哪？补救策略是什么？
   例：转化率虽然提升但客诉量也涨了 20%，正在排查原因

5. **下一步计划**：目标-差距-策略-资源
   例：下月 GMV 目标 5500 万，差距 1300 万，策略是加大直播投入，需要增加 2 名运营

没有的信息可以跳过，我不会编造数据。
```

## 报告模板

### 阿里周报模板

```markdown
# [团队/业务线] 周报 YYYY.MM.DD

## 核心结论

[一句话总结业绩完成情况 + 关键风险/价值观亮点]

## KPI 达成

| KPI | 目标 | 实际 | 达成率 | 较上周 |
|-----|------|------|--------|--------|
| [KPI1] | X | Y | Z% | +N% |
| [KPI2] | X | Y | Z% | +N% |

## 本周关键动作

- **[动作1]**：[具体做了什么] → 结果：[量化产出] → 目标差距从 X 缩小至 Y
- **[动作2]**：[具体做了什么] → 结果：[量化产出]

## 价值观实践

- **[价值观名称]**：在[具体场景]中，我选择了[动作]，放弃了[捷径/短期利益]，结果是[可验证的结果]

## 问题与复盘

- **目标差距**：[差距是什么，量化]
- **根因分析**：[为什么没达到]
- **补救策略**：[目标-差距-策略-资源]

## 下周计划

| 事项 | 目标 | 策略 | 所需资源 |
|------|------|------|---------|
| [事项1] | [目标] | [策略] | [资源] |
| [事项2] | [目标] | [策略] | [资源] |
```

- **结果**：[可验证的结果]

### [价值观2：具体名称]
- **场景**：[什么情况下]
- **选择**：[做了什么]
- **结果**：[可验证的结果]

## 组织贡献

- 团队建设：[具体贡献]

## 反空话规则 (Anti-Fluff Rules)

阿里风格特别禁止以下表达：

| 禁止 | 替换为 |
|------|-------|
| 拥抱变化 | 在[场景]中，面对[变化]，调整了[策略]，结果是[数据] |
| 客户第一（空喊） | 为了客户[具体需求]，放弃了[短期利益]，客户满意度从 X 到 Y |
| 做了大量努力 | 完成了 N 个[任务单位]，核心产出是[列举] |
| 赋能了XX团队 | 为XX团队提供了[具体支持]，帮助他们达成[量化结果] |
| 积极沟通拉通 | 与XX团队对齐了[具体事项]，结论是[决策]，落地时间[日期] |
| 打了一场漂亮的仗 | [项目名]达成目标[数据]，超出预期 X%，关键打法是[策略] |

### 阿里风格核心要求

1. **双轨并重**：业绩和价值观都要有事实支撑，缺一不可
2. **价值观要有"场景-选择-结果"**：不是喊口号，是在具体场景中做了什么选择
3. **目标-差距-策略-资源**：阿里的思考框架，每个问题都按这个结构分析
4. **复盘要有根因**：不是"下次注意"，是分析清楚为什么没做到
5. **用数据说话**：阿里文化强调"用事实和数据说话"，判断必须有依据
6. **自我要求高**：今天最好的表现是明天最低的要求——展现持续进步

## 质量检查

生成后自查：

- [ ] 业绩部分是否有明确的 KPI 目标值和达成率？
- [ ] 价值观部分是否有具体的"场景-选择-结果"事例？
- [ ] 复盘是否做了根因分析（不是简单归因"下次注意"）？
- [ ] 是否有"目标-差距-策略-资源"的分析框架？
- [ ] 是否避免了空喊价值观口号（每个价值观都有行为证据）？
- [ ] 数据是否来自用户提供的信息（没有编造）？
- [ ] 是否有禁用词表中的模糊表达残留？
