# 1:1 沟通准备 (One-on-One Prep)

继承 `manage-up-core` 五大原则。本技能解决的不是“帮你写一份汇报”，而是**帮你在 30 分钟 1:1 里把最值得说的话说出来**。

---

## 触发场景

- 用户要准备和直属上级、skip-level 的 1:1
- 用户想整理本周重点、阻塞、反馈、职业发展诉求
- 用户提到 "1:1 prep"、"prepare for my one-on-one"、"和老板一对一聊什么"

---

## 必填输入 (Required Inputs)

```
为了准备一场高质量的 1:1，请提供以下信息：

1. **这次 1:1 的对象是谁？**
   例：直属上级；skip-level；新老板

2. **你最近最重要的 2-3 件事是什么？**
   例：完成会员系统重构 80%；Q2 规划完成；带新人独立上线 2 个功能

3. **当前最需要讨论的问题/阻塞是什么？**
   例：第三方供应商卡住联调；团队缺 1 名前端；优先级反复变化

4. **你想从这次 1:1 得到什么？**
   例：要一个决策；争取资源；拿反馈；讨论职业发展

5. **有没有敏感话题？**（可选）
   例：工作负荷过高；晋升预期；跨团队合作问题

6. **时间长度和风格偏好？**（可选）
   例：30 分钟，想要简洁版；45 分钟，希望有开放讨论问题

没有的信息可以跳过，我不会编造数据。
```

## 报告模板

### 中文模板

```markdown
# 与 [对象] 的 1:1 准备

## 本次目标

[一句话：这次 1:1 最想达成什么]

## 优先讨论事项

1. **[事项 1]**
   - 现状：[事实 / 数据]
   - 为什么重要：[对项目 / 团队 / 你本人的影响]
   - 我希望得到的帮助：[具体请求 / 反馈 / 决策]

2. **[事项 2]**
   - 现状：
   - 为什么重要：
   - 我希望得到的帮助：

## 建议提问

- [向老板确认优先级 / 反馈 / 资源的具体问题]
- [一个开放问题]

## 如果时间不够，至少要讲这三句

- [最重要结论]
- [最重要风险或诉求]
- [最明确的请求]

## 会后跟进

- [需要记录的 action item]
```

### English Template

```markdown
# 1:1 Prep with [Manager]

## Goal for This Meeting

[One sentence: what you want to achieve]

## Priority Topics

1. **[Topic 1]**
   - Current state: [fact / data]
   - Why it matters: [impact]
   - What I need: [decision / feedback / support]

2. **[Topic 2]**
   - Current state:
   - Why it matters:
   - What I need:

## Questions to Ask

- [specific question about priorities / feedback / support]
- [one open-ended question]

## If Time Is Tight, Say These Three Things

- [most important conclusion]
- [most important risk or ask]
- [clearest next step]

## Follow-up

- [action item to capture after the meeting]
```

## 反空话规则 (Anti-Fluff Rules)

| 禁止 | 替换为 |
|------|-------|
| 想和老板聊聊最近的情况 | 这次 1:1 的目标是 [具体目标] |
| 最近做了很多事情 | 最近 2-3 件最重要的事是 [列出事实] |
| 有一些卡点 | 当前阻塞是 [具体问题]，影响 [量化后果] |
| 想听听反馈 | 我希望得到关于 [具体主题] 的反馈 |
| want to chat about a few things | goal: align on [specific topic] |
| have some blockers | blocker: [specific issue], impact: [quantified consequence] |

---

## 写作规则

1. **1:1 不是周报复述**：只挑最需要老板参与的事
2. **每个话题都要有“想得到什么”**：否则只是发泄，不是管理沟通
3. **敏感话题提前结构化**：事实、影响、建议，别情绪化开场
4. **给开放问题留空间**：1:1 不是你单向汇报
5. **准备“时间不够版本”**：很多 1:1 会被压缩

---

## 质量检查

- [ ] 本次 1:1 目标是否明确？
- [ ] 是否只保留了最重要的 2-3 个话题？
- [ ] 每个话题是否都有现状、影响和诉求？
- [ ] 是否包含至少一个高质量提问？
- [ ] 是否准备了时间不够时的短版本？
- [ ] 是否避免了模糊表达和情绪化描述？
